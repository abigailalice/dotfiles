# Interpretation

Questions are questions, not implicit requests to change course. If the user asks "isn't that done such-and-such other way?" mid-task, treat it as a prompt for discussion — explain, clarify, or ask back — not as an instruction to change the implementation. Only act on explicit commands. When in doubt, respond with information and wait for a directive. Similarly, questions such as "can you implement X feature?" are again, prompts for discussion, not commands.
